Latar Belakang

Mengapa Kredit Mikro Butuh SPK?

Masalah yang Ada
Penilaian Subjektif

Penilaian kredit oleh manusia bersifat subjektif dan rentan bias — dua petugas bisa menghasilkan keputusan berbeda untuk nasabah yang sama.

Data Bersifat Fuzzy / Ambigu

Tidak ada batas tegas antara "pendapatan cukup" dan "pendapatan kurang". Data keuangan nasabah bersifat kontinu dan relatif.

Solusi & Tujuan
Logika Fuzzy sebagai Solusi

Mampu menangani ketidakpastian linguistik secara matematis dan transparan, menghasilkan keputusan yang konsisten dan dapat dijelaskan.

Pertanyaan Penelitian
  • Bagaimana membangun SPK kredit mikro yang objektif?
  • Mana yang lebih akurat: Mamdani atau Sugeno?
  • Bagaimana perbandingannya dengan model ML & DL?

Fuzzifikasi

Fungsi Keanggotaan — Trapesium & Segitiga

Diimplementasikan from scratch menggunakan NumPy — tanpa library fuzzy pihak ketiga.

Trapesium (trapmf)

Parameter: a, b, c, d
Digunakan untuk himpunan ekstrem (kiri dan kanan). Nilai 1.0 di antara b dan c.

Segitiga (trimf)

Parameter: a, b, c
Digunakan untuk himpunan tengah (moderat). Puncak tepat di b = 1.0.

Pendapatan:
  Rendah = Trap(0, 0, 30K, 50K)
  Sedang = Segt(30K, 65K, 100K)
  Tinggi = Trap(80K, 100K, 1.5M, 2M)
5 / 18
Fuzzifikasi

Konversi Nilai Crisp → Derajat Keanggotaan μ

Setiap nilai input dikonversi ke derajat keanggotaan (μ) di masing-masing himpunan. Nilainya antara 0.0 – 1.0, dan bisa berada di lebih dari satu himpunan sekaligus.

Contoh Input Default
Pendapatan : $45,000
Utang : $1,500
Suku Bunga : 12%
Keterlambatan : 15 hari
Frek. Telat : 4 kali
Nilai μ = 1.0 berarti anggota penuh. μ = 0 berarti bukan anggota himpunan tersebut.

Proses Inferensi

Firing Strength & Agregasi MAX

Alur Evaluasi Aturan
Fuzzifikasi — hitung μ tiap himpunan untuk 5 variabel input
Evaluasi 34 Aturan — firing strength = MIN semua μ yang terlibat
α = min(μ_inc, μ_debt, μ_ir, μ_del, μ_num)
Agregasi MAX — gabungkan aturan per konsekuensi
u_baik = max(α seluruh aturan BAIK)
Kirim ke Defuzzifikasi — u_baik, u_standar, u_buruk
Hasil Contoh (Input Default)

Aturan aktif: 3 dari 34  ·  Dominan: BAIK
u_baik paling besar → sistem condong ke "Kredit Layak"

Model Pendukung

Regresi Linear & ANN/MLP — Referensi Informatif

Regresi Linear (ML)
  • 5 fitur → target numerik (Poor=0, Standard=1, Good=2)
  • Dilatih pada 80% × 60.000 sampel bersih
  • Hasil prediksi: Standar (raw score: 0.999)
ANN/MLP Classifier (DL)
  • Arsitektur: 5 → 64 → 32 → 16 → 3 (ReLU + Early Stopping)
  • Input: StandardScaler normalization
  • Prob. Buruk: 25.7% · Standar: 50.8% · Baik: 23.5%
⚠️ Catatan: Nilai ML & DL di atas hanya sebagai pembanding informatif. Keputusan akhir sistem ditentukan 100% oleh mesin inferensi Fuzzy Logic.

Preprocessing Data — Pembersihan & Seleksi Fitur

Langkah Preprocessing
  1. Bersihkan karakter asing dari kolom numerik (regex: [^\d.\-])
  2. Clip nilai negatif → Delay & Num Delayed ≥ 0
  3. Drop baris yang mengandung NaN setelah konversi
  4. Mapping label → Poor=0, Standard=1, Good=2
  5. Drop duplikat
  6. Filter outlier bisnis (Income ≤ 1.5M, Rate ≤ 35%)
  7. Sampling 60.000 baris acak (random_state=42)
  8. Train/Test Split → 80 : 20

Analisis

Mamdani vs Sugeno — Kelebihan & Kekurangan

Mamdani — Centroid
Kelebihan
  • Output kontinu → presisi lebih tinggi
  • Defuzzifikasi mudah diinterpretasikan
  • Sesuai desain rule linguistik
Kekurangan
  • Komputasi berat — integrasi numerik iteratif
  • Tidak efisien untuk real-time / embedded
Sugeno — Weighted Average
Kelebihan
  • Komputasi sangat cepat — aritmatika sederhana
  • Efisien untuk dataset besar & real-time
  • Tidak perlu MF output
Kekurangan
  • Output diskret — nilai singleton tetap
  • Konstanta singleton ditentukan manual

Kesimpulan: Kedua metode dijalankan paralel sebagai cross-validation. Mamdani unggul dalam presisi, Sugeno unggul dalam kecepatan. Selisih skor < 5 poin → keputusan konsisten.

Mengapa akurasi hanya ~58%?
Fuzzy logic berbasis aturan manusia, bukan di-training dari data. Parameter MF bisa dioptimasi lebih lanjut menggunakan GA atau PSO.

Mengapa 34 aturan?
Dirancang dari kombinasi linguistik 5 variabel × 3 himpunan yang merepresentasikan skenario kredit paling umum.
